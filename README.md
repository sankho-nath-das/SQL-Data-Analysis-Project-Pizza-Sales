# Pizza Sales Analysis using SQL

## Project Overview
This project analyzes a pizza sales dataset using SQL to uncover key business insights such as revenue trends, customer ordering behavior, and product performance. The analysis focuses on transforming raw transactional data into meaningful insights that can support data-driven decision making.

## Dataset
The dataset contains pizza sales transactions including:
- Orders
- Order details
- Pizza types
- Pizza categories and prices

These tables were joined and analyzed using SQL queries.

## Tools Used
- MySQL
- SQL (Joins, Aggregations, Window Functions)
- Data Analysis Techniques

## Key Analysis
The project answers several business questions including:
- Total number of orders placed
- Total revenue generated from pizza sales
- Highest priced pizza
- Most commonly ordered pizza size
- Top 5 most ordered pizza types
- Category-wise pizza sales distribution
- Hourly order distribution
- Average number of pizzas ordered per day
- Revenue contribution by pizza category
- Cumulative revenue trends over time

## Key Insights
- Total orders analyzed: **21,350**
- Total revenue generated: **~$817K**
- Peak order time: **12 PM – 1 PM (Lunch hours)**
- The **Classic category** contributes the highest revenue
- Average daily demand is **~138 pizzas**

## Business Recommendations
- Increase staff and inventory during lunch hours
- Promote underperforming categories
- Implement upselling strategies to increase order value
- Introduce targeted promotions during slow hours

## Project Goal
The goal of this project is to demonstrate how SQL can be used to analyze transactional datasets and extract actionable business insights.
